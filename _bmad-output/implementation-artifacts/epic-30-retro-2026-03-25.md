## What Went Well

### Conditional story gating saved 3 stories
Stories 30.6-30.8 were gated on profiling results from 30.5. The NO-GO was clean and data-driven — RocksDB is the bottleneck, not transport. This validated the "profile before optimizing" pattern and avoided 3 stories of unnecessary work.

### Parallel SDK agents remain effective
Both rounds (initial implementation and Cubic fix round) used 5 concurrent agents. First round completed in ~6-10 min each. Pattern continues to scale.

### Stories 30.3-30.4 were surgical
Zero test failures, zero clippy issues, zero debug struggles. The decomposition (refactor scheduler command first, then fix the handler) was clean. Story 30.4 touched only 4 files and delivered the throughput improvement.

### Throughput improvement was significant
3.1x improvement from eliminating per-message round-trips. Latency stayed well under NFR targets (p50 0.33ms, p99 0.49ms).

## What Didn't Go Well

### PROCESS FAILURE: External SDK PRs opened but not reviewed/merged
Story 30.2 launched 5 parallel agents that opened PRs, and the story was marked "done." But nobody checked CI status or waited for Cubic. All 5 had CI failures (stale fila-server binary from before proto changes) and 11 Cubic findings total. Required a second round of 5 fix agents. Root cause: the CLAUDE.md rule said "open PRs" but didn't gate story completion on CI + Cubic + merge.

### Previous retro action items not executed
Epic 29 retro had 4 action items. Two were marked "encode" but never actually written to files. 25% follow-through — worst score in 30 epics. Root cause: the retro workflow's Step 11 ("execute encoded changes") wasn't enforced strictly enough.

### CI test failure treated with empty retry
Story 30.1 had a CI failure (`sharded_broker_create_and_enqueue`) that was "fixed" with an empty retry commit instead of investigating the root cause. Violates the "never label flaky without investigating" rule.

### Clippy cascade from Cubic fix
PR #112 Cubic fix #2 removed a call site, making `parse_grpc_code` and `parse_stream_error` dead code. This caused a Clippy failure requiring a second fix commit. Minor but avoidable.

## Key Takeaways

1. **Conditional story gating works.** Profiling-driven NO-GO saved 3 stories. The pattern of gating later stories on measurement is validated across Epics 22-24, 29, and 30.

2. **Opening PRs is not completing them.** The external SDK PR lifecycle had a gap — agents opened PRs and moved on without waiting for CI or Cubic. Now encoded in CLAUDE.md AND execute-epic step-05.

3. **Retro "encode" decisions must be executed in the same session.** Epic 29's 25% follow-through proved that deciding to encode is not the same as encoding. Retro workflow Step 11 now has a verification gate.

4. **Documentation ships with code changes, not as a follow-up.** Docs went stale across 3 files after Epic 30's API changes. Now enforced at planning, dev, review, PR, and retro stages across 5 workflows.

5. **Process rules belong in workflows, not CLAUDE.md or memory.** Workflows are structurally enforced; CLAUDE.md rules are easily overlooked. Retro encode-or-drop routing now explicitly prefers workflows.
