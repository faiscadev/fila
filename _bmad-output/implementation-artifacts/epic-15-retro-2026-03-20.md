## Successes

1. **Fifteenth consecutive epic at 100% completion** — 3/3 stories delivered. 15 epics, every one at 100%. Full auth layer: mTLS transport, API key authentication, per-queue ACLs.

2. **Cubic earned its keep in Story 15.2** — 8 real findings across 2 review cycles: security bypass scope (bootstrap problem), `poll_ready` contract violation, tower layer ordering, silent `MetadataValue` failure, missing name validation, silent deserialization, test specificity, stale doc comment. Security code rightly attracts higher scrutiny.

3. **Bootstrap design improved under review pressure** — Evolved from "CreateApiKey bypasses auth" → `bootstrap_apikey` config/env var → `CallerKey` enum. Each iteration more secure and ergonomic. Classic security design progression.

4. **81 new tests (316+ → 397)** — TLS integration tests (valid cert, no cert, wrong CA), auth tests (5 scenarios), ACL tests (8 scenarios including pattern matching, superadmin bypass, cross-queue isolation), plus unit tests across auth module, config parsing, pattern matching.

5. **Zero debug struggles across security-critical code** — Auth middleware, tower layers, TLS handshakes, certificate validation — all shipped on first pass without debugging sessions.

6. **Backward compatibility preserved** — `Option<TlsParams>` (no TLS section = disabled), `Option<AuthConfig>` (no auth section = disabled). Existing deployments, tests, and SDKs work unchanged.

## Challenges

1. **External SDKs NOT updated** — All 3 story files explicitly note external SDKs as out of scope. Only Rust SDK (`fila-sdk`) and CLI got TLS + API key support. The 5 external SDKs (Go, Python, JS, Ruby, Java) cannot use mTLS or API key auth. This is the biggest gap from Epic 15.

2. **Cubic review density in 15.2 was high (8 findings)** — Security code attracts more review attention (correctly). The initial auth middleware implementation had rough edges around tower service contracts and error handling edge cases. Quality improved through the epic — 15.3 had fewer findings.

## Key Insights

1. **Security code attracts and deserves higher review scrutiny** — 8 Cubic findings in 15.2 vs typical 0-3. Auth middleware bugs have outsized impact (bypass = full access). The review process correctly applied more pressure here.

2. **Bootstrap problem is universal in auth systems** — The design evolution (bypass → config bootstrap → CallerKey enum) is textbook. First approach convenient but insecure; final approach both secure and ergonomic. Good pattern for future auth work.

3. **SDK feature gap accumulates silently** — Epic 9 shipped 5 external SDKs. Epics 14 (clustering) and 15 (auth) added server features. The SDKs are now behind. Without a plan to update them, users try Fila, find the SDK doesn't support auth, and bounce.

4. **`Option<T>` for config sections eliminates impossible states** — Both `TlsParams` and `AuthConfig` use `Option<T>` (presence = enabled). No `enabled: bool` field that can disagree with the config data. Pattern validated across 2 stories.
