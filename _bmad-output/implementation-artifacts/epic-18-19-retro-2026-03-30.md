## Challenges

### Agent Declares "Blocked" Prematurely

The dev agent stops at the first obstacle without trying alternatives:
- **Flamegraph skipped** — claimed "macOS SIP blocks dtrace" without trying `sample` (no sudo), `dtrace` (with sudo), or Instruments.app. Lucas had to push multiple times before it got done.
- **Prior art not checked** — correct `skip(self, request)` pattern already existed in git history. Agent implemented from scratch instead of checking prior art.

### Wrong Recommendations Propagate Across Artifacts

`skip_all` recommendation from 18.1 research doc flowed into 18.2 story spec, then into 18.2 implementation. Three artifacts contaminated by one wrong suggestion. Lucas caught it during review and pointed to the archive branch with the correct pattern.

### Unnecessary Backward Compatibility

Story 19.2 added legacy `message: Option<Message>` fallback fields in ClusterRequest — following the Raft backward compat rule. But Fila is pre-alpha with no users; the legacy fields added code complexity with zero value. Rule was followed mechanically without considering context.

### Competitive Bench CI Waste

`bench-competitive.yml` runs ~13 min on every PR producing artifacts nobody reads. Should be `workflow_dispatch` only.

## Key Insights

1. **Agent "blocked" is usually "didn't try hard enough"** — not a model knowledge gap, not a missing rule. The agent has the tools and instructions; it just stops too early. Workflow enforcement (try 3+ alternatives, list what you tried) is the fix.

2. **Review is the real quality gate** — Lucas caught: error metadata gap, continuation frames, batch naming, backward compat waste, `skip_all` vs `skip(self, request)`, missing flamegraph deliverable. Multiple design-level catches per epic.

3. **Naming matters** — if batch-of-1 is the only path, calling it "batch" implies an alternative exists. `handle_enqueue_batch` renamed to `handle_enqueue`. Code should read as if the design was always multi-item.

4. **Recommendations must be verified before propagating** — research doc recommendations should be cross-checked against actual codebase before becoming story ACs. Wrong inputs produce wrong outputs.
